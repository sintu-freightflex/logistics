## 2. Database Overview
| Attribute | Value |
|---|---|
| DBMS | MySQL 8.0 (InnoDB engine) |
| Character Set | `utf8mb4` (full Unicode, 4-byte) |
| Collation | `utf8mb4_unicode_ci` |
| Timezone | UTC stored in DB; converted to local in application layer |
| Primary Keys | `CHAR(36)` UUID generated by Python `uuid.uuid4()` |
| Soft Delete | `deleted_at DATETIME NULL DEFAULT NULL` — records never hard deleted |
| Audit Columns | `created_at DATETIME DEFAULT CURRENT_TIMESTAMP`, `updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` |
| ORM | SQLAlchemy 2.0 with `pymysql` driver |
| Migrations | Alembic — versioned, reversible |

## 4. SQLAlchemy Model Definitions

### 4.1 `users` table
```python
class User(Base):
    __tablename__ = "users"
    __table_args__ = {"mysql_engine": "InnoDB", "mysql_charset": "utf8mb4"}

    id              = Column(CHAR(36), primary_key=True, default=lambda: str(uuid4()))
    full_name       = Column(String(200), nullable=False)
    email           = Column(String(320), nullable=False, unique=True)
    phone           = Column(String(20), nullable=False)
    password_hash   = Column(String(60), nullable=False)
    role            = Column(Enum("DRIVER","HAULIER","FIRM","ADMIN"), nullable=False)
    status          = Column(Enum("INACTIVE","ACTIVE","SUSPENDED"), nullable=False, default="INACTIVE")
    profile_complete= Column(Boolean, nullable=False, default=False)
    verified        = Column(Boolean, nullable=False, default=False)
    avg_rating      = Column(DECIMAL(3, 2), default=0.00)
    completed_jobs  = Column(Integer, default=0)
    location_lat    = Column(DECIMAL(10, 7))
    location_lng    = Column(DECIMAL(10, 7))
    bank_account_id = Column(String(100))
    push_token      = Column(String(500))
    created_at      = Column(DateTime, default=lambda: datetime.now(timezone.utc))
    updated_at      = Column(DateTime, default=lambda: datetime.now(timezone.utc),
                             onupdate=lambda: datetime.now(timezone.utc))
    deleted_at      = Column(DateTime, nullable=True)

Index("idx_users_email", User.email)
Index("idx_users_role_verified", User.role, User.verified)
Index("idx_users_location", User.location_lat, User.location_lng)
```

**MySQL DDL:**
```sql
CREATE TABLE users (
    id              CHAR(36)        NOT NULL,
    full_name       VARCHAR(200)    NOT NULL,
    email           VARCHAR(320)    NOT NULL,
    phone           VARCHAR(20)     NOT NULL,
    password_hash   VARCHAR(60)     NOT NULL,
    role            ENUM('DRIVER','HAULIER','FIRM','ADMIN') NOT NULL,
    status          ENUM('INACTIVE','ACTIVE','SUSPENDED') NOT NULL DEFAULT 'INACTIVE',
    profile_complete TINYINT(1)     NOT NULL DEFAULT 0,
    verified        TINYINT(1)      NOT NULL DEFAULT 0,
    avg_rating      DECIMAL(3,2)    DEFAULT 0.00,
    completed_jobs  INT             DEFAULT 0,
    location_lat    DECIMAL(10,7),
    location_lng    DECIMAL(10,7),
    bank_account_id VARCHAR(100),
    push_token      VARCHAR(500),
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at      DATETIME,
    PRIMARY KEY (id),
    UNIQUE KEY uk_users_email (email),
    KEY idx_users_role_verified (role, verified),
    KEY idx_users_location (location_lat, location_lng)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 5. MySQL-Specific Considerations

### 5.1 UUID vs Auto-increment
- UUIDs (`CHAR(36)`) chosen over `AUTO_INCREMENT` for security (non-enumerable IDs) and future distributed-DB compatibility.
- Slight index size penalty vs INT is acceptable at Phase 1 scale.
- Phase 2 option: store as `BINARY(16)` with `BIN_TO_UUID()` / `UUID_TO_BIN()` for tighter storage.

### 5.2 JSON Columns
- MySQL 8.0 native `JSON` type used for `checklist_data`, `condition_photo_urls`, `fleet_details`, `notifications.data`.
- Queried via `JSON_EXTRACT()` or SQLAlchemy's `JSON` column with Python dict/list access.
- No JSONB (PostgreSQL-only) — MySQL JSON is stored as binary tree internally.

### 5.3 Boolean Storage
- MySQL stores `BOOL/BOOLEAN` as `TINYINT(1)`. SQLAlchemy `Boolean` column maps transparently.

### 5.4 Timezone
- All `DATETIME` columns store UTC. MySQL server `time_zone` is set to `'+00:00'`.
- Application converts UTC ↔ local timezone in the API response layer.

### 5.5 Spatial Queries (No PostGIS)
- Proximity matching uses the Python **Haversine formula** (`matching_service.py`) rather than PostGIS (PostgreSQL-only).
- MySQL 8.0 has native spatial types (`POINT`, `ST_Distance_Sphere`) — Phase 2 optimisation.

### 5.6 Full-text Search
- Job search by address/goods type can use MySQL `FULLTEXT` index on `pickup_address`, `drop_address`, `goods_type` in Phase 2.

## 6. Connection Configuration (SQLAlchemy)
```python
# app/database.py
DATABASE_URL = "mysql+pymysql://user:password@rds-host:3306/freightflex?charset=utf8mb4"

engine = create_engine(
    DATABASE_URL,
    pool_size=10,           # persistent connections
    max_overflow=20,        # burst connections
    pool_pre_ping=True,     # ping before reuse (handles MySQL 8h idle timeout)
    pool_recycle=3600,      # recycle connections every hour
    connect_args={"ssl": {"ca": "/path/to/rds-ca.pem"}},  # RDS SSL
```
